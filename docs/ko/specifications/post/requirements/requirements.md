# Post 도메인 — 요구사항

## 시스템 개요

Post 도메인은 Notion 데이터베이스에서 가져온 블로그 포스트 표시를 담당합니다. 포스트 목록, 상세 렌더링, 태그 기반 필터링, 포스트 간 네비게이션을 포함합니다. 태그도 이 도메인에 포함됩니다.

### 이해관계자

| 역할 | 관심사 |
|------|--------|
| 독자 | 블로그 포스트 탐색, 필터링, 읽기 |
| 블로그 소유자 | Notion에서 글 작성, 블로그에서 렌더링 확인 |
| AI 에이전트 | FSD 패턴에 따라 포스트 관련 기능 유지보수 |

## 기능 요구사항

### FR-POST-01: 포스트 목록 표시

| 항목 | 값 |
|------|---|
| ID | FR-POST-01 |
| 우선순위 | 높음 |
| 설명 | Notion에서 가져온 공개 포스트를 발행일 내림차순으로 그리드 표시 |

**완료 기준**:
- 상태가 "공개"인 포스트만 표시
- "날짜" 기준 내림차순 정렬
- 각 포스트 카드: 커버 이미지, 제목, 발행일, 태그, AI 요약 (있을 경우)
- 반응형 그리드: 1열 (모바일) → 4열 (데스크톱)

### FR-POST-02: 포스트 상세 페이지

| 항목 | 값 |
|------|---|
| ID | FR-POST-02 |
| 우선순위 | 높음 |
| 설명 | `/posts/[slug]`에서 리치 포맷의 전체 Notion 페이지 콘텐츠 렌더링 |

**완료 기준**:
- URL은 slugify된 제목 사용 (`github-slugger`)
- `react-notion-x`로 코드 블록, 컬렉션, 수식 포함 Notion 콘텐츠 렌더링
- 테마 인식 렌더링 (다크/라이트 모드)
- 모든 공개 포스트에 대해 정적 파라미터 사전 생성 (SSG)
- 포스트 데이터에서 메타데이터 (제목, 설명) 생성

### FR-POST-03: 태그 기반 필터링

| 항목 | 값 |
|------|---|
| ID | FR-POST-03 |
| 우선순위 | 중간 |
| 설명 | 포스트 목록 페이지에서 클라이언트 사이드 다중 선택 태그 필터링 |

**완료 기준**:
- 태그 필터 사이드바에 Notion 데이터베이스의 모든 태그 표시
- 여러 태그 선택 가능 (OR 조건 — 선택된 태그 중 하나라도 있으면 매칭)
- 페이지 새로고침 없이 클라이언트 사이드 필터링
- 매칭되는 포스트 없을 때 빈 상태 표시
- 공개 포스트에 존재하는 태그만 활성으로 표시

### FR-POST-04: 포스트 네비게이션

| 항목 | 값 |
|------|---|
| ID | FR-POST-04 |
| 우선순위 | 중간 |
| 설명 | 상세 페이지에서 관련 포스트 간 네비게이션 |

**완료 기준**:
- 최대 4개 관련 포스트 표시 (태그 공유, 발행일 가장 가까운 순)
- 포스트 콘텐츠 하단에 2열 그리드로 표시
- 각 관련 포스트에 아이콘, 제목, 태그 표시

### FR-POST-05: 추천 포스트 (홈 페이지)

| 항목 | 값 |
|------|---|
| ID | FR-POST-05 |
| 우선순위 | 중간 |
| 설명 | 홈 페이지에서 전체 포스트를 그리드로 표시 |

**완료 기준**:
- 홈 페이지에 모든 공개 포스트와 함께 `FeaturedPosts` 컴포넌트 표시
- ISR 재검증 180초

## 비기능 요구사항

### NFR-POST-01: 성능

- ISR 재검증: 목록 페이지 180초, 상세 페이지 설정 가능
- Notion API 호출에 `nextServerCache()` 서버 사이드 캐싱
- 빌드 시 모든 포스트 상세 페이지 정적 생성

### NFR-POST-02: 데이터 무결성

- Post, Tag 도메인 모델은 타입 가드와 protected 생성자 사용
- 유효하지 않은 Notion 응답은 설명적 에러 throw
- Slug 매핑은 slugify된 제목과 원시 Notion ID 모두 지원

## 제약사항

| 유형 | 제약 |
|------|------|
| 기술 | Notion 데이터베이스 속성은 한국어 이름 사용 (제목, 날짜, 상태, Tags) |
| 기술 | 두 개의 Notion 클라이언트 필요: 쿼리용 공식, 렌더링용 비공식 |
| 아키텍처 | Post 엔티티: `src/entities/post/`, Post 기능: `src/features/post/` + `src/features/tag/` |
