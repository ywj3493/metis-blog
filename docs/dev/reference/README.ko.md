# 참조 문서

## 목적

이 디렉토리는 서드파티 통합, API, SDK 가이드 및 외부 시스템 작업 시 Claude와 개발자가 필요로 할 수 있는 기타 기술 문서에 대한 외부 참조 자료를 포함합니다.

## 포함 항목

**포함**:

- 외부 API 문서 (Notion, OpenAI 등)
- SDK 통합 가이드
- 서드파티 서비스 설정 지침
- 인증 및 자격 증명 관리 가이드
- 벤더별 모범 사례
- 속도 제한 및 할당량 정보
- 웹훅 및 콜백 명세

**포함하지 않음**:

- 프로젝트별 코드 또는 아키텍처 (`docs/specifications/` 사용)
- 개발 정책 및 규칙 (`docs/policy/` 사용)
- 이슈 계획 문서 (`docs/issue/` 사용)
- 내부 구현 세부사항

## 참조 자료 추가

### 추가 시점

다음의 경우 참조 문서를 추가하세요:

1. 새로운 외부 서비스 또는 API 통합 시
2. 복잡한 서드파티 설정 절차 문서화 시
3. 중요한 벤더별 지식 캡처 시
4. 공식 문서에서 명확하지 않은 컨텍스트 제공 시

### 추가 방법

1. **적절한 이름의 파일 생성**: 설명적인 kebab-case 이름 사용
   - 예시: `notion-api-reference.md`, `openai-models-guide.md`, `vercel-deployment.md`

2. **출처와 날짜 포함**: 정보의 출처와 시점 명시
   - 예시 헤더:
     ```markdown
     # Notion API 참조

     출처: https://developers.notion.com/reference
     최종 업데이트: 2024-01-15
     ```

3. **프로젝트 관련 세부사항에 집중**: 전체 문서를 복사하지 말고 중요한 내용 추출
   - 관련 엔드포인트
   - 인증 요구사항
   - 속도 제한
   - 이 프로젝트에서 사용되는 일반적인 패턴

4. **기존 참조 업데이트**: API 변경에 따라 문서를 최신 상태로 유지

## 현재 참조 자료

_아직 없음. 외부 서비스와 통합할 때 필요에 따라 참조 자료를 추가하세요._

## 유지보수

### 휴먼 개발자의 책임

- 외부 서비스 통합 시 새 참조 자료 추가
- API 변경 시 참조 업데이트
- 오래된 문서 제거
- 관련 시 Claude에게 이 문서들의 컨텍스트 제공

### Claude의 역할

- 통합 작업 시 참조 자료 **읽기**
- 이슈 문서와 코드 주석에서 이 문서들 **참조**
- 정보가 누락된 경우 새 참조 자료 **요청**
- **생성하지 않음**: Claude는 명시적으로 요청받지 않는 한 참조 문서를 생성하지 않음

## 효과적인 참조 문서 작성 팁

1. **간결하게**: 필요한 것만 추출, 전체 벤더 문서를 복제하지 않음
2. **공식 문서 링크**: 항상 권위 있는 출처 링크 포함
3. **주의사항 기록**: 명확하지 않은 동작과 엣지 케이스 문서화
4. **예시 포함**: 이 프로젝트에서 API가 어떻게 사용되는지 보여줌
5. **모든 것에 날짜 표시**: API는 변경되므로 참조가 생성/업데이트된 시점 기록
